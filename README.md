# Fortimove 운영 시스템 (Fortimove-OS)

**주의: 본 저장소는 웹 개발 리포지토리가 아닙니다.**
이곳은 Fortimove 전체 운영 시스템, SOP(표준작업절차), 브랜드 가이드라인 등을 중앙 집중화하여 관리하는 **비즈니스 문서 저장소**입니다.

## 1. 폴더 구조 및 역할

명확한 관리 원칙을 위해 아래 구조를 유지합니다.

*   `docs/`: 실무 기준서, 가이드라인(SOP), 마케팅/기획 등 사람이 주로 읽는 비즈니스 운영 문서.
*   `prompts/`: 반복 업무 및 콘텐츠 생성에 활용할 AI 프롬프트 템플릿 보관.
*   `templates/`: 고객 응대, 발주서 등 반복 실무용 각종 서식 및 양식 보관.
*   `.claude/`: Claude AI가 프로젝트 컨텍스트를 파악할 때 가장 먼저 참고해야 하는 프로젝트 특화 지침서 (`CLAUDE.md`).
*   `.agents/rules/`: AI 에이전트(Cursor, Roo Code 등)가 코드 작성 및 답변 시 범용적으로 적용해야 할 일관된 규칙.

## 2. 문서 권장 읽기 순서

저장소를 파악하기 위해 아래 순서대로 문서를 읽는 것을 권장합니다.

1.  `README.md` (본 문서: 전체 구조 및 금지사항 파악)
2.  `docs/business-goals.md` (단기/중기/장기 비즈니스 목표)
3.  `docs/brand-context.md` (브랜드 분리 및 사업 전략 원칙)
4.  `docs/sourcing-sop.md`, `docs/product-registration-sop.md`, `docs/cs-sop.md` (실무 파이프라인)

## 3. 저장소 제출 금지 항목 (Not Allowed)

이 저장소는 **순수 비즈니스 운영 문서**로만 기능해야 합니다. 따라서 아래 항목은 절대 이곳에 넣지 마십시오.

*   웹 개발 코드 (웹사이트, 앱 등을 구동하는 `src/`, `components/`, HTML/CSS/JS 파일 전체)
*   실행 가능한 스크립트 도구 및 스크래핑 봇 실행 파일 (`.py`, `.sh` 등 개발 리포지토리 성격의 코드)
*   임시 테스트 파일 및 쓰레기 값 문서 (`test.md`, `dummy.txt` 등)
*   개인정보가 포함된 실제 고객의 주문 내역 DB 파일이나 정산 엑셀 원본 파일

## 4. 브랜드 구조 (Brand Context) 요약

*   **Fortimove Global (현재 실행 브랜드)**: 단순 구매대행, 신규 아이템 소싱 테스트. 매출 및 시장 데이터 확보(단기적 캐시카우).
*   **Fortimove (장기 메인 브랜드)**: 웰니스·헬스케어 플랫폼. 데이터가 검증된 핵심 상품의 권리(PB/독점)를 확보하여 쌓아가는 브랜드(장기적 자산).

## 5. 문서 작성 및 버전 관리 원칙

*   **한국어로 작성할 것.**
*   **실무형으로 작성할 것.**
*   **추상적 표현은 배제할 것.**
*   **복붙 가능한 완성형**: 대표 및 실무자가 바로 [복사/붙여넣기] 해서 사용할 수 있는 완성형을 지향할 것.
*   **문서 버전 관리**: 주요 정책(SOP 등) 변경 시 사유와 함께 버전을 업데이트하여 기록할 것.
