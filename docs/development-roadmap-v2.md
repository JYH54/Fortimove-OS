## Phase 5: 안정화 + 실전 검증 (1-2주)

> 새 기능보다 **기존 시스템을 실전에서 쓸 수 있는 수준**으로 올리는 것이 우선

| # | 작업 | 현재 | 목표 | 영향 |
|---|------|------|------|------|
| 5-1 | **PM Agent 에이전트간 데이터 자동 전달** | 수동 연결 | 소싱→마진→등록 완전 자동 체인 | 사람 개입 최소화 |
| 5-2 | **상세페이지 편집기 실전 테스트** | 기능 구현됨 | 실제 상품 3개로 E2E 테스트 | 등록 워크플로우 완성 |
| 5-3 | **Sentinel 크롤러 정확도 향상** | 노이즈 있음 | 수원/경기 지원사업 정밀 타겟팅 | 자금줄 확보 |
| 5-4 | **Docker 통합 배포** | 시스템별 개별 실행 | docker-compose 하나로 전체 기동 | 운영 편의성 |
| 5-5 | **대시보드 네비게이션 통합** | 페이지별 산재 | 메인→COO/리뷰/리디자인/에디터 원클릭 | UX |

## Phase 6: 매출 파이프라인 완성 (2-3주)

> 소싱부터 등록까지 **사람이 최소한으로 개입하는 자동화 파이프라인** 완성

| # | 작업 | 설명 | 비즈니스 임팩트 |
|---|------|------|----------------|
| 6-1 | **Daily Scout → PM Agent 자동 연결** | 트렌드 스캔 결과가 자동으로 소싱 에이전트에 투입 | 상품 발굴 자동화 |
| 6-2 | **상세페이지 → pick2sell 내보내기** | 최종 이미지를 pick2sell 형식(개별 JPG)으로 자동 패키징 | 등록 시간 단축 |
| 6-3 | **SEO 상품명/태그 자동 생성 → CSV** | pick2sell 기본정보 탭에 바로 붙여넣기 가능한 형식 | 등록 시간 단축 |
| 6-4 | **옵션/판매가 자동 계산** | 4개국 마진 에이전트 결과 → pick2sell 판매가 탭 형식 | 가격 실수 방지 |
| 6-5 | **Korean Law MCP → 소싱 에이전트 강화** | 법령 검증 자동 호출 안정화 (현재 70%) | 컴플라이언스 |

## Phase 7: 비즈니스 인텔리전스 고도화 (3-4주)

> COO Agent + Sentinel을 **실시간 경영 의사결정 도구**로 진화

| # | 작업 | 설명 |
|---|------|------|
| 7-1 | **매출 트래킹 연동** | 스마트스토어/쿠팡 판매 데이터 자동 수집 → COO 대시보드 |
| 7-2 | **경쟁사 모니터링** | 키프리스 상표권 + 동일 상품 가격 추적 |
| 7-3 | **Sentinel 자동 신청서 초안** | 지원사업 공고 → 신청서 양식 자동 작성 |
| 7-4 | **COO 주간 리포트** | 매주 월요일 자동 경영 보고서 → Slack/이메일 |
| 7-5 | **위닝 상품 재주문 알림** | 재고 소진 예측 → 벤더 재발주 타이밍 제안 |

## 우선순위 판단 기준

```
돈이 되는 것 > 돈을 지키는 것 > 편의성 > 확장성
```

1. **Phase 5** → 지금 만든 것을 실제로 쓸 수 있게 (안정화)
2. **Phase 6** → 매출이 나오는 파이프라인 완성 (돈)
3. **Phase 7** → 더 똑똑하게 운영 (지능)
4. **Phase 8** → 커졌을 때 대비 (확장)
